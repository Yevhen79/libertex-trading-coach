# Libertex «Trading Buddy» — продуктовый драфт (Hackathon)

> Собственная версия Guardian Angel для терминала Libertex. Поведенческий AI-коуч, встроенный в терминал: после каждой сделки — короткий персональный фидбэк, раз в N сделок — развёрнутый AI-разбор с оценками.
> Автор направления: Евгений Шакотько (PM MetaTrader, Libertex). Референс: CPattern Guardian Angel (demo). Deep research → `../research/01-deep-research.md`.

---

## 0. TL;DR для питча Хакатона

**Проблема:** в CFD 70–80% ритейл-трейдеров сливают депозит и уходят. Основная причина — не рынок, а **поведение**: торговля без стопов, оверлеверидж, revenge-trading, панические выводы после просадки. Мы теряем клиента до того, как он «научился».

**Решение:** встроенный поведенческий коуч, который в реальном времени показывает трейдеру **зеркало его собственных привычек**, подкрепляет хорошее, мягко переформулирует плохое и предупреждает о риске *до* margin call. Не даёт торговых советов (комплаенс-safe).

**Почему это деньги (industry benchmarks у CPattern/AvaTrade):** +42% средний депозит, +19–33% сделок/объёма, доп. депозиты 53% vs 41%, 95% юзеров говорят что фича влияет на их решения. Бьёт ровно в ретеншн + оборот + депозиты.

**Почему это Хакатон-buildable:** `rules-engine (мгновенные тексты) + Claude LLM (нарративный разбор) + виджет в терминале + feedback-loop`. MVP на исторических сделках демо-аккаунтов собирается за 2–3 дня на Claude Code.

**Рабочее имя:** «**Trading Buddy**» / «**Наставник**» / «**Libertex Coach**» (варианты — см. §8). Далее в тексте — *Buddy*.

| | CPattern Guardian Angel | Libertex Trading Buddy (наш) |
|---|---|---|
| Интеграция | Плагин/EA поверх MT4/5 | **Нативно в проприетарном терминале Libertex + приложении**, потом MT4/5 |

| Стоимость | Лицензия SaaS | Свой продукт → нет per-seat лицензии, полный контроль roadmap |

**Позиционный оффер клиенту:** *«Libertex — единственный, кто торгует на твоей стороне: показывает не только рынок, но и тебя самого»*.

## 2. Как это работает — архитектура (Хакатон-версия)

```
┌─────────────────────────────────────────────────────────────┐
│ 1. TRADE EVENTS (open/close)  ← Libertex terminal / MT bridge │
│         │                                                     │
│         ▼                                                     │
│ 2. BEHAVIORAL ENGINE  (считает метрики по истории клиента)    │
│    • avg profit / avg loss        • % сделок со стоп-лоссом    │
│    • размер позиции / free margin  • диверсификация активов    │
│    • длительность удержания        • win/loss streak           │
│    • оверлеверидж, revenge-паттерн  • комиссии «впустую»        │
│         │                    │                                │
│         ▼                    ▼                                │
│ 3a. RULES ENGINE          3b. LLM REVIEW (Claude)             │
│    мгновенный текст          раз в N сделок: нарратив + scores │
│    (шаблон + подстановка)    (structured prompt, см. §5)       │
│         │                    │                                │
│         ▼                    ▼                                │
│ 4. WIDGET в терминале (карточка) + push в приложение          │
│         │                                                     │
│         ▼                                                     │
│ 5. FEEDBACK LOOP  👍/👎, ⭐ rating → тюнинг + аналитика        │
└─────────────────────────────────────────────────────────────┘
```

**Разделение труда LLM vs rules:**
- **Rules engine** — мгновенные пост-трейд карточки. Дёшево, быстро, детерминировано, комплаенс-контролируемо. 90% показов.
- **LLM (Claude)** — только периодический «AI Review» (раз в 10 сделок / раз в день / по кнопке). Дорого, но глубоко и «человечно». Тексты проходят через guardrail-фильтр (§6).

## 3. Продуктовая логика: как каждая механика бьёт в 3 метрики

> Явное сопоставление «фича → метрика», чтобы на питче и в PRD не было воды.

### 3.1. РЕТЕНШН (удержание)

| Механика | Как работает | Почему удерживает |
|---|---|---|
| **Ранний risk-алерт до margin call** | «Твоя позиция по GOLD использует 78% свободной маржи — при движении на 1.2% ты близок к margin call» | Клиент не «взрывается» неожиданно → не уходит в обиде; чувствует, что брокер на его стороне (рост доверия — главный драйвер adoption в кейсе AvaTrade) |
| **Reframing просадки** | «Ты потерял 103$, это на 24$ больше твоего среднего убытка. Но за 10 сделок ты в целом +28$» | Снижает панику и revenge-trading → меньше эмоциональных сливов → дольше живёт |
| **Ежедневный AI-Review как повод открыть приложение** | Готовый разбор «ждёт тебя» → habit loop | Daily active use ↑ → ретеншн ↑ |
| **Прогресс/обучение** | «Твоя дисциплина выросла с 5 до 7 за 2 недели» | Ощущение роста = причина остаться |

### 3.2. ТОРГОВЫЙ ОБОРОТ

| Механика | Как работает | Почему растёт оборот |
|---|---|---|
| **Позитивное подкрепление хороших сделок** | «Good work! Ты закрыл в плюс и со стопом — так держать» | Оператное подкрепление → повторение дисциплинированной торговли (+19% сделок в бенчмарке) |
| **Streak / серии** | «3 сделки подряд со стоп-лоссом 🔥 — не разрывай серию» | Игровая петля мотивирует продолжать |
| **Re-engagement дормантов** | Push: «Тебя ждёт свежий AI-разбор твоей последней сессии» | Возврат спящих → доп. объём |
| **Мягкая диверсификация** | «Ты торгуешь только EUR/USD. Твои прибыльные сделки были и на GOLD — посмотри аналитику по нему» (без совета «купи») | Расширение набора инструментов → больше сделок |

### 3.3. ДОПОЛНИТЕЛЬНЫЕ ДЕПОЗИТЫ

| Механика | Как работает | Почему доносят деньги |
|---|---|---|
| **Доверие через заботу** | Вся механика reframing + risk-алертов | Доверие → готовность фандить (депозиты +42%, re-deposit 53% vs 41% в бенчмарке) |
| **Нарратив «ты улучшаешься»** | AI-Review показывает рост скоров | Клиент верит в свой прогресс → доносит капитал, чтобы «масштабировать» |
| **Прозрачная стоимость недо-капитализации** | «Маленькая свободная маржа заставляет тебя закрывать рано: средняя сделка живёт 10 мин» (факт, не призыв) | Клиент сам приходит к решению пополнить (комплаенс: не «пополни», а «вот факт») |

## 4. Тексты уведомлений (готовый копирайтинг)

> Формат карточки-виджета: **иконка настроения + заголовок + 1–2 предложения с ПОДСТАВЛЕННОЙ СТАТИСТИКОЙ КЛИЕНТА + строка про AI-Review + 👍/👎**.
> Переменные в `{фигурных скобках}`. Тон: на «ты», поддерживающий, без нравоучений, без торговых советов.

### 4.1. После закрытия в убыток (reframing)

**RU:**
> 😕 **Небольшая просадка**
> Ты закрыл {symbol} с убытком {loss}$. Это на {delta}$ больше твоего среднего убытка ({avg_loss}$). Бывает — важно, что за последние {n} сделок ты в целом {net_sign}{net}$.
> 🤖 *AI-разбор будет готов через {k} сделок.*

**EN:**
> 😕 **Small drawdown**
> You closed {symbol} at −{loss}$, {delta}$ more than your average loss ({avg_loss}$). It happens — over your last {n} trades you're still {net_sign}{net}$.
> 🤖 *Your AI review will be ready in {k} more trades.*

### 4.2. После закрытия в плюс (подкрепление)

> ✅ **Хорошая работа!**
> Прибыль {profit}$ по {symbol}{with_sl_note}. {reinforce_line}
> 🤖 *Твой AI-разбор готов — открыть.*

где:
- `{with_sl_note}` = « — и со стоп-лоссом, отлично» / «»
- `{reinforce_line}` = «Твой средний профит {avg_profit}$ — ты в тренде.» или «Это выше твоего среднего ({avg_profit}$) 👏»

### 4.3. Открытие сделки без стоп-лосса (риск, мягко)

> ⚠️ **Сделка без стоп-лосса**
> Ты открыл {symbol} без SL. За последние 30 дней {sl_free_pct}% твоих убыточных сделок были без стопа. Просто держим в курсе — решение за тобой.

### 4.4. Оверлеверидж / близко к margin call (ретеншн-критично)

> 🛟 **Высокая нагрузка на маржу**
> Позиция {symbol} использует {margin_pct}% твоей свободной маржи. При движении на {distance}% против тебя возможен margin call. Это не совет — просто хотим, чтобы ты знал заранее.

### 4.5. Revenge-trading паттерн

> 🧘 **Пауза?**
> Ты открыл {count} сделки за {minutes} минут после убытка. Часто такие «отыгрыши» у тебя закрываются в минус ({revenge_winrate}% в плюс). Никто не торопит.

### 4.6. Позитивный streak (геймификация оборота)

> 🔥 **Серия {streak}**
> {streak} сделки подряд со стоп-лоссом. Твоя дисциплина на этой неделе — {discipline_score}/10.

### 4.7. Концентрация в одном активе (мягкая диверсификация)

> 🎯 **Ты почти весь в {symbol}**
> {concentration_pct}% твоих сделок за месяц — это {symbol}. Для сравнения, твои сделки по {other_symbol} закрывались в плюс в {other_winrate}% случаев. Аналитику по рынкам найдёшь в разделе Insights.

### 4.8. Re-engagement дормантного клиента (push)

> 👋 **Тебя ждёт свежий разбор**
> Мы подготовили AI-обзор твоей последней торговой сессии: стиль, риск, что улучшить. Занять 30 секунд.

## 5. AI Trading Review — LLM-слой (Claude)

Триггер: каждые 10 закрытых сделок / кнопка / ежедневно. Модель получает **агрегированные метрики** (не сырые PII), возвращает structured-нарратив по фиксированному шаблону.

### 5.1. Промпт (каркас для Claude)

```
Роль: ты — поведенческий трейдинг-коуч Libertex. Ты анализируешь ПРИВЫЧКИ трейдера,
НЕ даёшь торговых рекомендаций (не называй что покупать/продавать, без прогнозов цены).
Тон: поддерживающий, на «ты», конкретный, честный. Опирайся ТОЛЬКО на переданные метрики.

Данные последних 10 сделок (агрегаты):
{metrics_json}   // total, avg_profit, avg_loss, largest_win/loss, sl_usage_pct,
                 // avg_volume, avg/max/min_duration, per_asset_winrate, streaks,
                 // прошлый период для сравнения

Верни строго 6 секций + оценки:
1. Trading Style Profile — тип трейдера (scalper/day/swing) по длительности и частоте.
2. Trading Parameters Summary — маркированный список ключевых чисел.
3. Performance: Improvement or Deterioration — vs прошлый период, самый прибыльный актив.
4. Risk-Taking Patterns — плечо, стопы, размер к марже. Без осуждения.
5. Handling of Greed & Fear — по паттернам (revenge, ранние закрытия, догоны).
6. Learning Curve & Progress — что растёт, 1 конкретная привычка на след. период.
Затем Scores (0–10): Consistency, Discipline, Rational Trading — с 1 строкой обоснования.
Заверши дисклеймером: «AI может ошибаться. Это разбор поведения, не инвестсовет.»
```

### 5.2. Пример готового вывода (RU)

> **Разбор последних 10 сделок**
>
> **1. Твой стиль.** Ты swing-трейдер: средняя сделка живёт 10 минут, ловишь среднесрочные откаты. Работает эффективно 🚀
>
> **2. Ключевые цифры.**
> • Сделок: 10 • Лучшая: EUR/USD (+54$) • Худшая: BTC/USD (−103$)
> • Средний профит: 4.2$ • Средний убыток: 60.5$ • Со стоп-лоссом: 0%
> • Средняя длительность: 10 мин (макс 12 / мин 2)
>
> **3. Динамика.** Результаты неровные vs прошлый период. Самый прибыльный актив — EUR/USD. Средний профит слегка снизился ⭐
>
> **4. Риск.** Риск умеренный, но плечо высокое и **стопов не было ни в одной сделке** — это твоя главная зона роста.
>
> **5. Жадность и страх.** В целом решения рациональные, без явных эмоциональных всплесков 🕊
>
> **6. Прогресс.** Ты стал фокусироваться на EUR/USD — самом прибыльном активе, это зрелость. **Привычка на следующие 10 сделок: поставь стоп-лосс хотя бы в половине сделок.**
>
> **Оценки:** Consistency **7/10** · Discipline **6/10** · Rational Trading **8/10**
>
> _AI может ошибаться. Это разбор поведения, не инвестиционный совет._
>
> ⭐⭐⭐⭐⭐ Оцени разбор

## 6. Комплаенс-guardrails (не пропустить!)

Регуляторно это должно оставаться **поведенческим зеркалом, а не советником** (как у CPattern — «does not make suggestions about placing orders»).

**Красные линии (запрещено в любых текстах и в LLM-выводе):**
- ❌ «Купи / продай / закрой / открой {symbol}» — никаких направленных призывов.
- ❌ Прогнозы цены («BTC пойдёт вверх»).
- ❌ Обещания прибыли / «безопасно».
- ❌ Призывы «пополни депозит» / «увеличь плечо» — только факты, вывод делает клиент.

**Обязательно:**
- ✅ Дисклеймер в каждом AI-Review.
- ✅ Post-processing фильтр над LLM-выводом (regex + классификатор на запрещённые интенты) до показа.
- ✅ Формулировки риска — нейтральные, информирующие («просто держим в курсе, решение за тобой»).
- ✅ Обработка данных без PII на LLM-слое (агрегаты, а не паспорт клиента).
- ✅ Согласие пользователя (opt-in) + возможность отключить/настроить типы уведомлений.

## 7. Гейминг-слой (усилитель оборота и ретеншна)

- **3 живых скора:** Consistency / Discipline / Rational Trading (0–10), тренд-стрелка неделя-к-неделе.
- **Streaks:** «N сделок подряд со стопом», «M дней с положительным разбором».
- **Badges/levels** (как Edgewonk «level up»): «Risk Manager», «Diversifier», «Cool Head».
- **Личный рекорд, не сравнение с другими** (self-competitive — так у CPattern, безопаснее психологически и юридически).
- Всё это — вокруг **дисциплины**, а не P&L, чтобы не провоцировать овертрейдинг ради лидерборда.

## 9. Scope Хакатона (что реально собрать) + Claude Code-план

### MVP (демо на Хакатон)
1. **Данные:** взять историю закрытых сделок демо/тест-аккаунтов (CSV/экспорт из терминала или MT-бридж).
2. **Behavioral engine:** посчитать 8 метрик (§2) на Python.
3. **Rules engine:** 8 шаблонов уведомлений (§4) с подстановкой.
4. **LLM Review:** Claude по промпту §5 → structured-разбор + скоры.
5. **Guardrail-фильтр** над LLM-выводом (§6).
6. **Виджет:** веб-карточка (React-компонент) в стиле терминала Libertex, навигация `< n of N >`, 👍/👎, ⭐.
7. **Demo-скрипт:** проиграть 10 реальных сделок → показать 10 карточек + финальный AI-Review.

### Stretch
- Push в мобильное приложение, re-engagement сценарий.
- Feedback-loop пишет в аналитику; A/B-каркас.
- Мультиязычность (RU/ES/PT) — Claude генерит локализованные варианты.

### Claude Code-velocity (как строить)
- «Направь Claude Code собрать behavioral engine из CSV сделок с 8 метриками» → 0.5 дня.
- «Сгенерь rules-engine с 8 шаблонами и unit-тестами подстановки» → 0.5 дня.
- «Собери React-виджет-карточку в стиле Libertex + Storybook» → 1 день.
- «Промпт + guardrail-фильтр + golden-тесты на запрещённые интенты» → 0.5 дня.
- Итого **~2.5–3 дня** до кликабельного демо.

## 10. Как мерить успех (чтобы не верить маркетингу вендора)

- **Основной KPI:** A/B — когорта с Buddy vs контроль. Метрики: D30-retention, доп. депозиты (%, сумма), сделок/юзер, объём, доля сделок со стопом (proxy на «здоровье» клиента = LTV).
- **Гипотеза для защиты:** ждём хотя бы **половину** бенчмарк-эффекта CPattern (напр. +10–20% доп. депозитов, +5–10% ретеншн) — уже сильный ROI.
- **Guardrail-метрика:** НЕ должен расти овертрейдинг/сливы. Следим за средним убытком и margin-call rate — они должны падать, а не расти.
- **Adoption:** opt-in rate, % включивших уведомления, 👍/👎 ratio, ⭐ среднее (у CPattern 4.6/5 — планка).
