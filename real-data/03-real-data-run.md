## 1. Как получаем историю сделок (ответ на вопрос)

Веб-терминал Libertex грузит закрытые сделки чистым JSON-эндпоинтом (сессия — по кукам залогиненного пользователя):

```
GET https://app.libertex.org/spa/report/closed-positions
    ?page=1&pageSize=200&order=CloseTime&orderDir=desc&searchPhrase=
    [&startCloseDate=<epoch_ms>]   // фильтр периода; без него — «за всё время»
```

Ответ:
```
{ status, result: { closed:[…], summary:{totalCount,totalSummInv,totalEquityInv}, instruments:[…], date, lastCloseDate }, messages, codeVersion }
```

Поля сделки (`result.closed[i]`):
`ticket, symbol, alias, direction (growth=buy | reduction=sell), mult, sumInv, equityInv, startRate, closeRate, startTime(ms), closeTime(ms), stopLossPrice (null = без стопа), takeProfitPrice`

Производные (наш движок):
- **P&L = `equityInv − sumInv`**, `P&L% = P&L / sumInv × 100` (сверено с UI: $9.59 − $10 = −$0.41 = −4.10% ✓)
- **Есть стоп = `stopLossPrice != null`** → метрика «% сделок со стопом» напрямую
- Длительность = `(closeTime − startTime)/60000` мин

**Пути интеграции для продакшена (по возрастанию усилий):**
1. Тот же внутренний эндпоинт `report/closed-positions` — если Buddy живёт внутри веб-терминала (server-to-server, по account id).
2. Кнопка **«Скачать»** на странице закрытых → выписка (для ручного MVP/бэктеста).
3. Для MT4/MT5-аккаунтов — история через отчёт терминала / investor-пароль / MT5-gateway (у тебя как PM MetaTrader есть доступ).

> На этом счёте есть ещё демо (`kosmos`) с **1265 сделками** — куда более богатый датасет, если нужен объёмный прогон.
